## Опис

- `server.cfg` - це файл конфігурації сервера, який дозволяє змінювати всі види налаштувань вашого sa-mp сервера.
- Цей файл є обов'язковим для кожного сервера і повинен знаходитись у каталозі сервера поруч із серверною програмою (samp-server.exe).

## Консоль

| Параметр | Тип | Значення за замовчуванням | Тільки для читання | Правило | Ефект
| ------------- | ------ | -------------------------- | --------- | ---- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| echo | string | Executing Server Config... | Yes | No | Те, що samp-server.exe друкує в консолі сервера при виконанні server.cfg. Не потрібно нічого змінювати, оскільки консоль бачитимете лише ви.                                                                                          |
rcon_password | string | changeme | No | Ні | Ні | Пароль, який використовується для адміністрування сервера і використання віддаленої консолі (rcon). Переконайтеся, що ви змінили його на щось, що важко зламати, щоб інші не змогли отримати контроль над вашим сервером. Ваш сервер НЕ запуститься, якщо changeme буде паролем RCON! |
| rcon | bool | 1 | Ні | Ні | Вмикає або вимикає функцію [Віддалена консоль](RemoteConsole). Встановіть 1, щоб увімкнути, або 0, щоб вимкнути.

## Сценарії

| ------------------- | ------ | --------------------- | --------- | ---- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
gamemode(n) (N) (t) | string | gamemode0 grandlarc 1 | No | Ні | Ні | Файл .amx у теці gamemodes, який сервер має використовувати як ігровий режим. (n) - номер ігрового режиму, (N) - назва ігрового режиму без розширення .amx, і (t) - кількість разів, яку потрібно зіграти в цей режим, перш ніж переходити до наступного ігрового режиму.                                                                                                                                                                                                                                                        |
| filterscripts (N) | рядок | | Так | Ні | Файл .amx у теці filterscripts, який сервер повинен використовувати як фільтрскрипт. (N) - це назва фільтрскрипту без розширення .amx. Фільтрскрипти - це скрипти, які виконуються у фоновому режимі гри. Вони призначені для додавання додаткових можливостей на сервер без редагування ігрового режиму. Це дуже корисно, якщо ви хочете перенести певну властивість на декілька ігрових режимів. Якщо ви хочете завантажити декілька скриптів фільтрів, додайте їх підряд, розділяючи пробілом, наприклад filterscripts скрипт1 скрипт2. |
| плагіни (N) | рядок | | Так | Ні | Файл .dll або .so у теці плагінів, який сервер повинен використовувати для запуску як плагін. У Windows (N) - це назва плагіна без розширення .dll. Однак, в Linux розширення .so є обов'язковим! Плагіни - це скрипти, призначені для покращення ігрових режимів та скриптів фільтрів. Якщо ви хочете завантажити кілька плагінів, вкажіть їх поспіль, розділяючи пробілом, наприклад, plugins plugin1 plugin2.                                                                                                                     |

## Примітки

- Значення, позначені як «Тільки для читання» (як показано у `/rcon varlist`), не можна змінювати під час виконання. Всі інші значення можна (тимчасово) змінити, передавши їх команді SendRconCommand.
- Значення, позначені як «Rule» (як показано у `/rcon varlist`), відображаються у браузері сервера у розділі Rules.
- Скрипти не обов'язково повинні знаходитися у теках gamemodes або filterscripts. Інформація у файлі server.cfg - це шлях, а отже, можна використовувати «...».
